Basic usage

Below are the few steps you need to follow to use the plugin.

  1. Add FormaJS stylesheet inside the HTML document <head>.
  2. Add jQuery library script and forma.js before the closing </body> tag.
  3. Create your form — don't forget to add .forma as a class attribute.
  4. Initialize the plugin and specify additional options (see more details below).
  5. Customize and style you form, either by using the pre-made CSS forma.css and tweak the :root variables, use the forma-base.css to build a custom layout, or create your own stylesheet form scratch (see more details below).

Starter template

Use the code below as your starting point, for testing or development.

<!doctype html>
<html lang="en">
    <head>
        <meta charset="utf-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no" />
        <link href="favicon.ico" rel="icon" />
        <title>FormaJS. - Creating dynamic and interactive forms.</title>
        <!-- Include FormaJS stylesheet. -->
        <link href="path/to/css/forma.css" rel="stylesheet" />
    </head>
        <!-- Add your form structre, class="forma" is required if you don't specify other container! -->
        <form class="forma">
            <label>
                <span>Text</span>
                <div class="fieldset">
                    <input type="text" />
                </div>
            </label>
            <button type="submit">Submit</button>
        </form>
        <!-- Load up the jQuery library first. -->
        <script src="https://code.jquery.com/jquery-3.4.1.slim.min.js"></script>
        <!-- Include FormaJS plugin script. -->
        <script src="path/to/js/forma-0.0.1.min.js"></script>
        <script>
            $.forma();
        </script>
    </body>
</html>

Form structure

There is a certain structure you need to follow when you build up your forms.

  1. Each field (row) need to be inside a <label> tag.
  2. Use <span> tag for the text (this where all the validation/data content will be appended).
  3. Lastly, put your form elements insde <div class="fieldset">...</div>. You can have multiple elements inside this div.

* Note: the submit button doesn't need to use this structure.

<form class="forma">
    <label>
        <span>Text</span>
        <div class="fieldset">
            <input type="text" />
        </div>
    </label>
    <button type="submit">Submit</button>
</form>

Custom options

When you initialize your form you can specify several different options. Below are the default values for all the options.

$.forma({
    container: '.forma',
    manual: false,
    show: false,
    auto: false,    
    support: [] // Not implemented yet.
});

container - Form target.

  • Usually defined with class attribute but you can also use id or data attributes as well. Useful when you have multiple forms on the same page.
  • Either string or HTML element; both formats are accepted '.forma' and $('.forma').
  • Default value is '.forma'.

manual - Add an open/close state button for each form field, if set as true.

  • By default to navigate between fields the user is required to use the tab key.
  • Must be a boolean type; accepted values true or false.
  • Default value is false.

show - Open all form fields on load, if set as true.

  • By default only 1st field is opened and focused on page load.
  • Must be a boolean type; accepted values true or false.
  • Default value is false.

auto - Display additional instructions text, if set as true.

  • Show additionally auto-generated description and placeholder text, for each field, using the label text.
  • Must be a boolean type; accepted values true or false.
  • Default value is false.

support - Add support to additional form elements. Not implemented yet.

Currenlty supported form elements are:

  • input[type="text"]
  • input[type="email"]
  • input[type="password"]
  • input[type="url"]
  • input[type="number"]
  • input[type="submit"]
  • input[type="reset"]
  • select
  • textarea
  • button

Custom styling

The easiest way to customize the form feel and look is to change the root variables inside forma.css and add your own colors, fonts etc... Below you can see the default values.

:root {
    --font: 400 normal 16px/1.6 Monaco, Consolas, 'Andale Mono', 'Ubuntu Mono', monospace;
    --primary-color: #D65DB1;
    --secondary-color: #845EC2;
    --error-color: #FF6F91;
    --success-color: #008F7A;
    --warning-color: #FF9671;
    --info-color: #0081CF;
    --white-rgb-color: 250, 250, 250; /* #FAFAFA */
    --black-rgb-color: 25, 25, 25; /* #191919 */
}

Note: You can additionally take a look at the rest of the file and with some basic knowledge of CSS customize additionally the layout of your form.

Create your custom CSS

To do this take a look at the boilerplate CSS file forma-base.css. This will give you a head start to style and customize your form.

Here is link to the GitHub repo prod/forma-base.css.

Extending forma-base.css

You can easily extend the forma-base.css and modify your form layout. Take a look at the Register form example.

With only 1 additional CSS rule you can make your form to turn into, one at time field, and swich between fields with the tab key.

.forma label:not(.forma-open) {
    display: none;
}

Adding the following CSS will turn your form into 2 column layout.

.forma label {
    align-items: stretch;
    flex-direction: row;
}
.forma span {
    flex: 1 0 320px;
    justify-content: center;
}
.forma .fieldset {
    align-self: flex-end;
    flex: 1 1 100%;
}
.forma span.invalid:before {
    position: relative;
    right: auto;
}

Create your own stylesheet form scratch

This is the most advanced method to customize your forms. However, if you have done quite a bit of HTML/CSS you will be able to create your own style from scratch.

Take a look at both forma.css and forma-base.css first, there are some classes added via JavaScript that you may want to style and target.

Validation

There are two types of validations, built-in validations with generic messages and custom validations that use regular expressions.

Built-in validations

These will display based on the type of the input field; <input type="email" /> will require a valid email address and output the following message if email isn't correct format Hint: enter a valid email address.

You can also add additional attributes to most form elements for validation, for example:

  • required;
  • minlength and maxlength;
  • min, max, and step (for numbers);
  • etc...
<label>
    <span>Username</span>
    <div class="fieldset">
        <input type="text" minlength="3" maxlength="32" required />
    </div>
</label>

* Example: Full built-in validation example here.

Custom validations

For real-world forms the built-in checks aren't enough so you would need to use some regular expression magic.

Not good at Regexp? Jump over to StackOverflow; 99% of the cases can be found there.

Use the pattern, and title (set your custom error message text) attributes.

MDN web docs reference can be found here.

<label for="password">
    <span>Password</span>
    <div class="fieldset">
        <input type="password" name="password" pattern="(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d]{8,}" 
            title="Password must have minimum 8 characters, at least one letter and one number" required />
    </div>
</label>
