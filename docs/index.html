Basic usage

FormaJS is very easy to setup and customize, however, it's flexible and could be used in many different scenarios.

Below are the things you need to keep in mind when use the script.

  • Add the variation of forma.css stylesheet inside the document <head> tag.
  • Add the forma.min.js script near the end of your document before the closing </body> tag.
  • Create your form. (see form structure details)
  • Initialize the script and specify additional options. (see custom options details)
  • Customize and style you form; either by using the pre-built stylesheets inside /dist/css and tweak the :root values or create your own stylesheet form scratch using the /dist/css/forma-boileplate.css as your starting point. (see styling details)

Starter template

Use the code below as your starting point, for testing or development.

<!doctype html>
<html lang="en">
 <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no" />
    <title>FormaJS - Create dynamic and interactive forms with a better UX.</title>
    <!-- Include FormaJS stylesheet variation. -->
    <link href="path/to/css/forma.css" rel="stylesheet" />
  </head>
  <body>
    <!-- Add your form structure, class="forma" is required if you don't specify other container! -->
    <form class="forma">
      <label>
        <span>Text</span>
        <div class="forma-fieldset">
          <input type="text" />
        </div>
      </label>
      <button type="submit">Submit</button>
    </form>
    <!-- Include FormaJS plugin script. -->
    <script src="path/to/js/forma.min.js"></script>
    <script>
      $forma();
    </script>
  </body>
</html>

Form structure

There is a certain structure you need to follow when you create your forms.

  1. Don't forget to add forma class to your target form tag & use div with forma-fieldset class to wrap your form elements.
  2. Each field (row) need to be inside a <label> tag.
  3. Use the <span> tag for text (this where all the validation/description & live input content is appended).
  4. Lastly, put your form elements inside a <div class="forma-fieldset">...</div> tag. You can have multiple elements inside.
<form class="forma">
  <label>
    <span>Text</span>
      <div class="forma-fieldset">
        <input type="text" />
      </div>
    </label>
  <button type="submit">Submit</button>
</form>

* Note: The submit button doesn't need to be inside a label tag. You can also use <input type="submit" /> as your submit button.

Custom options

When you initialize your form you can specify several different options. Below are the default values for all the options with detailed description for each one.

forma({
  container: '.forma',
  prefix: 'Enter ',
  suffix: '...',
  tab: false,
  auto: false,
  show: false,
  manual: false,
  submit: false,
  support: []
});

container - Form target.

  • Usually defined with class attribute but you can also use id or data attributes as well. (useful when you have multiple forms on the same page)
  • Must be of a string type.
  • Default value is '.forma'.

prefix - Text added before the auto-generated text.

  • Prepend text when you auto-generate additional text.
  • Must be of a string type.
  • Default value is Enter .

suffix - Text added after the auto-generated text.

  • Append text when you auto-generate additional text.
  • Must be of a string type.
  • Default value is ....

tab - Navigate between rows (fields) with tab key; make only the current element visible, if true.

  • Tab navigation will make visible only the current row (field) and turn off the rest.
  • Must be of a boolean type; accepted values true or false.
  • Default value is false.

auto - Display additional auto-generated text, if true.

  • Show additionally auto-generated description and placeholder text, for each field, using the label text and specified prefix and suffix.
  • Must be of a boolean type; accepted values true or false.
  • Default value is false.

show - Auto-open all form rows (fields) on load, if true.

  • By default only 1st field is opened and focused on page load.
  • Must be of a boolean type; accepted values true or false.
  • Default value is false.

manual - Add an open/close state button for each form row (field), if true.

  • By default to navigate between fields the user is required to use the tab key.
  • Must be of a boolean type; accepted values true or false.
  • Default value is false.

submit - Disable submit button, if true.

  • Disable submit button until all validations have passed.
  • Must be of a boolean type; accepted values true or false.
  • Default value is false.

support - Add additional element support.

Must be of an array type. You can add other form element support by passing another types.

Script supported form elements:

  • input
  • select
  • textarea
  • button
  • input[type="text"]
  • input[type="email"]
  • input[type="password"]
  • select[type="select-one"]
  • textarea[type="textarea"]
  • button[type="submit"]

* Hint: You will get a console.error message if the form element isn't supported.

Styling

The easiest way to customize the form is to change the :root values that come with all the pre-built forma.css files. Add your own colors, fonts etc...

Below you can see the default values included in all pre-built stylesheets inside /dist/css.

@import url('https://fonts.googleapis.com/css?family=Source+Sans+Pro');
:root {
  --font-mono: 400 normal 16px/1.6 'Monaco', 'Consolas', 'Andale Mono', 'Ubuntu Mono', monospace;
  --font-google: 400 normal 16px/1.6 'Source Sans Pro', sans-serif;
  --primary-color: #d65bd1;
  --secondary-color: #845ec2;
  --error-color: #ff6f91;
  --success-color: #008f7a;
  --warning-color: #ff9671;
  --info-color: #0081cf;
  --white-rgb-color: 250, 250, 250; /* #fafafa */
  --black-rgb-color: 25, 25, 25; /* #191919 */
  --space: 1rem;
}

Note: You can additionally take a look at the stylesheet files inside /dist/css and with some basic knowledge of CSS customize additionally the layout of your form.

Create your custom CSS

To do this take a look at the boilerplate CSS file /dist/css/forma-boilerplate.css. This will give you a head start to style and customize your form.

Here is link to the GitHub repo /dist/css/forma-boilerplate.css.

Extending forma.css

You can easily extend the forma.css and modify your form layout. Take a look at the Register form example.

With only 1 additional CSS rule you can make your form to turn into, one at time field, and switch between fields with the tab key.

.forma label:not(.forma-open) {
  display: none;
}

Create your own stylesheet form scratch

This is the most advanced method to customize your forms. However, if you feel comfortable with HTML/CSS you will be able to create your own styles from scratch easily.

Take a look at both /dist/css/forma.css and /dist/css/forma-boilerplate.css first; there are some additional classes added via JavaScript that you may want to style as well.

Validation

There are two types of validations; built-in validations with generic messages and custom validations that use regular expressions. (by utilizing the pattern and title attributes)

Built-in validations

These will display based on the type of the input field; for example, <input type="email" /> will require a valid email address and output the following message if the email address isn't a correct format Enter a valid email address.

You can also add additional attributes to most form elements for validation, for example:

  • required;
  • minlength and maxlength;
  • min, max, and step (for numbers);
  • etc...
<label for="username">
  <span>Username</span>
  <div class="forma-fieldset">
    <input type="text" name="username" minlength="3" maxlength="32" required />
  </div>
</label>

* Example: Form with built-in validations can be found here.

Custom validations

For real-world forms the built-in checks aren't enough so you would need to use some regular expression magic.

Use the pattern, and title attributes to set your custom error message text. More information can be found at MDN Web Docs.

<label for="password">
  <span>Password</span>
  <div class="forma-fieldset">
    <input type="password" name="password" pattern="(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d]{8,}" 
      title="Password must have minimum 8 characters, at least one letter and one number" required />
  </div>
</label>

* Example: Form with custom validations can be found here.

* Hint: Not good wit regexp? Jump over to StackOverflow; 99% of the use cases can be found there.
